# 📝 Neovim 設定・主要プラグイン・キーバインド詳細

このドキュメントでは，[modules/neovim.nix](../modules/neovim.nix) を通じて管理されている Neovim の基本設計，独自マッピング，および強力なプラグインエコシステムについて詳細に解説します．

---

## ⚙️ 基本設定 (General Options)

効率的なコーディングとモダンな視覚効果を実現するための基本オプション設計（[vim-options.lua](../modules/nvim/lua/vim-options.lua)）です．

* **タブ・インデント幅**: `expandtab`，`tabstop=4`，`shiftwidth=4` (スペース4つのクリーンなインデント)．
* **相対行番号 (`relativenumber`)**: 現在行からの相対距離で番号を表示し，`10j` などのVim移動手数を一瞬で測ることができます（`number` も有効なため，現在行のみ絶対行数が表示されます）．
* **クリップボード連携 (`clipboard=unnamedplus`)**: OSのシステムクリップボードとNeovimのレジスタが自動同期され，`y` でコピーしたテキストをブラウザや他のアプリへそのまま貼り付け可能です．
* **カーソルライン・カラムハイライト (`cursorline`，`cursorcolumn`)**: 現在カーソルがある行と列が十字状にハイライトされ，現在位置を一瞬で見失うのを防ぎます．
* **スペースLeader**: **`vim.g.mapleader = " "`** (スペースキーをLeaderキーとして機能させます)．
* **背景の透過処理**:
  - `ColorScheme` イベントをフックする自動コマンドにより，インストールしたテーマに関わらず **エディタ背景・フローティング窓・サインコラムの背景を完全に透過 (none)** させます．
  - WezTermの半透明背景と完全に調和し，デスクトップ壁紙が美しく透けて見えます．

## 🛠️ 強力な独自カスタムマクロ・コマンド

ユーザー様の執筆活動や開発作業を爆速にする，Luaで書かれた高度なオリジナルスクリプト機能です．

### 1. Zenn用スクリーンショット画像貼り付け機能
* **キーバインド**: **`<leader>ip`** (Paste Image)
* **挙動**: 
  1. クリップボードにある最新の画像ファイル（スクリーンショット等）を検出．
  2. 現在開いているZennブログ記事のファイル名（Slug）に基づいて `/images/ファイル名/image-日付.png` を自動生成し，プロンプトでファイル名変更を受け付けます．
  3. 自作スクリプト `get-clip-img` を内部で実行して画像を保存し，Markdown内に相対パスの画像埋め込みリンク `![](...)` を自動で即時挿入します．

### 2. コードカラーHTML自動保存コマンド
* **キーバインド**: **`<leader>th`** または **`:ToHtmlSave`** コマンド
* **挙動**: 現在アクティブなコードを，Neovimのシンタックスハイライト（カラー表示）をそっくりそのまま維持した状態で美しいHTMLファイルに変換し，現在のファイルと同じディレクトリに自動保存します．ソースコードを共有したりブログに載せる際に非常に重宝します．

## 📦 主要プラグインエコシステム (Plugin Ecosystem)

プラグインマネージャー **Lazy.nvim**（[init.lua](../modules/nvim/init.lua)）により，以下の最新鋭プラグインが高速に自動ロード・管理されます．

1. **`cyberdream.lua`**: 近未来的なサイバーパンク調の極めて美しいメインカラーテーマ．背景透過処理と美しく調和．
2. **`lsp-config.lua`**: 高度なコーディング支援（自動コード補完・エラー検出・定義ジャンプ）．
3. **`telescope.lua`**: 超高速あいまい検索ツール（ファイル名・コード内全文検索・Gitステータスをその場でポップアップ検索）．
4. **`neo-tree.lua`**: サイドバーにツリー状でディレクトリを表示するファイルエクスプローラー．
5. **`treesitter.lua`**: コードの構文解析を行い，非常に正確で鮮やかなシンタックスハイライトを実現．
6. **`dap.lua`**: エディタ内にブレークポイントを設定し，ステップ実行が可能な本格デバッガ．
7. **`flash.lua`**: 画面上のあらゆる場所に，わずか2〜3キーストロークでカーソルを瞬間移動させる超高速ナビゲーション．
8. **`snacks.lua`**: 起動画面（Dashboard）や通知，その他多数 of UIパーツを上品に美化・拡張する総合ユーティリティ．
9. **`which-key.lua`**: キー入力を一時停止した際，次に押せるショートカットキーと説明を画面下にポップアップ表示するカンニングペーパー機能．

## 🔌 各プラグイン個別キーバインド詳細

### 1. Snacks.nvim (新機能 & 便利ツール)

モダンなUIと高速なツール群を提供し，Git連携やフローティングターミナルなどの多機能な統合が行われています．

#### 📦 Git 操作 (LazyGit)
* `<Leader>lg` で起動した LazyGit ウィンドウ内では，**`Ctrl + G`** を入力することでGemini APIを使用した **AIコミットメッセージ自動生成** が実行できます．

| キーバインド | 動作内容 | 説明 |
| :--- | :--- | :--- |
| **`<Leader>lg`** | **Lazygit 起動** | フローティング窓でGit操作画面を開く |
| **`<Leader>lf`** | **ファイル履歴** | 今開いているファイルの変更履歴を表示 |
| **`<Leader>ll`** | **リポジトリログ** | プロジェクト全体のGitログを表示 |

#### 💻 ターミナル操作
| キーバインド | 動作内容 | 説明 |
| :--- | :--- | :--- |
| **`Ctrl + /`** | **ターミナル切り替え** | フローティングターミナルを表示/非表示 (トグル) |

### 2. Telescope (超高速あいまい検索)

プロジェクト内のファイルやテキストを縦横無尽に検索するためのキーバインドです．

| キーバインド | 動作内容 | 説明 |
| :--- | :--- | :--- |
| **`<Leader>ff`** | **ファイル検索** | ファイル名でプロジェクト内を検索 |
| **`<Leader>fg`** | **全文検索** | 文字列をファイルの中身から超高速検索 (grep) |
| **`<Leader>fb`** | **バッファ検索** | 現在開いているバッファ(ファイル)の一覧から検索 |
| **`<Leader>fh`** | **ヘルプ検索** | Neovimの公式ヘルプドキュメントから検索 |
| **`<Leader>fr`** | **最近のファイル** | 最近開いたファイルの履歴から検索 |
| **`<Leader>fw`** | **カーソル単語検索** | カーソル位置の単語でプロジェクト全体を検索 |
| **`<Leader>f.`** | **検索再開** | 直前の検索結果・検索画面をもう一度開く |

### 3. Neo-tree (サイドバーファイルエクスプローラー)

左側にツリー形式でディレクトリ構造を表示するエクスプローラー（**`<Leader>e`** で開閉）内の専用操作です．

| キーバインド | 動作内容 | 説明 |
| :---: | :--- | :--- |
| **`a`** | **新規作成** | ファイル (例: `name.lua`) またはフォルダ (例: `dir/`) を作成 |
| **`d`** | **削除** | 選択しているファイルまたはフォルダを削除 |
| **`r`** | **リネーム** | 選択しているファイルまたはフォルダの名前を変更 |
| **`l`** / **`Enter`** | **開く / 展開** | ファイルを開く，またはフォルダを展開する |
| **`h`** | **閉じる** | フォルダを閉じる |

### 4. セッション管理 (Persistence)

作業状態をディレクトリごとに自動で保存し，いつでも続きから再開するためのキーバインドです．

| キーバインド | 動作内容 | 説明 |
| :--- | :--- | :--- |
| **`<Leader>qs`** | **セッション復元** | 現在のディレクトリ用に保存されているセッションを読み込む |
| **`<Leader>ql`** | **前回復元** | 最後に使用していたセッションをそのまま読み込む |
| **`<Leader>qd`** | **保存しない** | 今回の終了時にセッションを自動保存しないようにマークする |

### 5. 開発支援・LSP機能

言語サーバー(LSP)と連携した高度なコード分析・編集支援ショートカットです．

| キーバインド | 動作内容 | 説明 |
| :---: | :--- | :--- |
| **`K`** | **ホバー表示** | カーソル下の型やドキュメントをポップアップ表示 |
| **`gd`** | **定義元へ移動** | 変数や関数の定義元ファイルへジャンプ |
| **`<Leader>ca`** | **コードアクション** | LSPによる修正提案・リファクタリングの実行 |
| **`<Leader>d`** | **診断表示 (行)** | 現在の行にあるエラーや警告の詳細を表示 |
| **`[d`** / **`]d`** | **診断箇所移動** | 前 / 次のエラー警告箇所へ素早くジャンプ |
| **`<Leader>q`** | **診断一覧** | 現在のファイル内にある全エラーをリスト表示 |
| **`gcc`** | **行コメント** | 現在の行をコメントアウト/解除 (Normalモード) |
| **`gc`** | **範囲コメント** | 選択範囲をコメントアウト/解除 (Visualモード) |

### 6. デバッグ (DAP - Debug Adapter Protocol)

コード内にブレークポイントを設定し，ステップ実行によるデバッグ作業を行うためのキーバインドです．

| キーバインド | 動作内容 | 説明 |
| :--- | :--- | :--- |
| **`<Leader>dc`** | **デバッグ開始 / 続行** | デバッグセッションを開始，または次のブレークポイントまで進む (`F5`) |
| **`<Leader>db`** | **BP切り替え** | 停止点(ブレークポイント)の設置/解除 (※`<Leader>b`でも実行可能) |
| **`<Leader>dn`** | **ステップオーバー** | 次の行へ進む (Next, `F10`) |
| **`<Leader>di`** | **ステップイン** | 関数やメソッドの内部に入る (Into, `F11`) |
| **`<Leader>do`** | **ステップアウト** | 現在の関数から抜け出す (Out, `F12`) |
| **`<Leader>dt`** | **デバッグ終了** | デバッグセッションを終了する (Terminate) |
| **`<Leader>du`** | **UI表示切り替え** | デバッグ専用パネル画面の表示/非表示をトグルする |

### 7. 高速瞬間移動 (Flash.nvim)

画面内に表示されているあらゆるテキストへ，最小限のキーストロークでカーソルを瞬間移動させます．

| キーバインド | 動作内容 | 説明 |
| :---: | :--- | :--- |
| **`s`** | **ジャンプ (Flash)** | 検索文字を入力すると画面内にラベルが表示され，その文字へ一瞬で飛ぶ |
| **`S`** | **範囲選択 (TS)** | 構文木 (Treesitter) を解析し，ブロックや関数単位でスマートに選択 |
| **`r`** | **リモート操作** | `y` (コピー) や `d` (削除) などのオペレータの対象を遠隔地から選択 |
| **`Ctrl + s`** | **検索トグル** | 標準の `/` 検索を実行中に，Flash機能のON/OFFをトグルする |

---

### 8. 編集・囲み操作 (Surround)

`nvim-surround` を用いた，カッコやクォート，HTMLタグといった囲み文字の追加・削除・変更操作です．
*(※`iw` は "Inner Word" (カーソル下の単語) を表します)*

| 操作キー | 動作内容 | 具体例 |
| :---: | :--- | :--- |
| **`ys + iw + "`** | **囲み文字を追加** | `hello` → `"hello"` |
| **`ds + (`** | **囲み文字を削除** | `(hello)` → `hello` |
| **`cs + ' + "`** | **囲み文字を変更** | `'hello'` → `"hello"` |
| **`cs + t + <p>`** | **HTMLタグの変更** | `<div>...</div>` → `<p>...</p>` |

### 9. バッファタブ (heirline.nvim)

開いているファイル (バッファ) を画面上部にタブとして表示します．2枚以上開いているときだけ表示され，配色は Matugen パレット (アクティブ = メインハイライト，非アクティブ = muted，未保存変更 = サブアクセントの●) に追従します．

| キーバインド | 動作内容 | 説明 |
| :---: | :--- | :--- |
| **`Tab`** | **次のタブへ** | 右隣のバッファタブに切り替える |
| **`Shift + Tab`** | **前のタブへ** | 左隣のバッファタブに切り替える |
| **`Q`** | **タブを閉じる** | 現在のバッファを閉じる (1キー版．標準のマクロ再生 `Q` は転用) |
| **`<Leader>bd`** | **タブを閉じる** | 現在のバッファを閉じる (`:bdelete`) |
| **`<Leader>bo`** | **他を全部閉じる** | 現在のバッファ以外 (未保存変更なし) をまとめて閉じる |

補足:
- WezTerm タブ / Starship と同じ「accent 色ブロック + ▶」デザイン (heirline による自作タブライン)
- タブを「開く」操作は通常のファイルオープンそのもの (`:e <file>`，Telescope の `<Leader>ff`，Neo-tree からの選択など)．開いたファイルが自動でタブに並びます．
- これはバッファ単位の表示です．表示部品は `plugins/heirline-tabline.lua` で完全自作しているため，形・色は自由に変更できます．
